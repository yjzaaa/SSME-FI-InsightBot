**⚠️ 时间范围查询重要注意事项：**

**日期字段带时分秒问题处理：**
由于数据库中的日期字段可能包含时分秒信息（如：'2025-07-15 14:30:22'），在进行日期范围查询时必须特别注意：

1. **结束日期处理：**
   - ❌ 错误写法：`CONVERT(datetime, CreateDate) <= '2025-07-15'`
   - ✅ 正确写法：`CONVERT(datetime, CreateDate) < '2025-07-16'` 或 `CONVERT(datetime, CreateDate) <= '2025-07-15 23:59:59'`

2. **标准日期范围查询模板：**
```sql
-- 查询2025年7月15日整天的数据
WHERE CONVERT(datetime, CreateDate) >= '2025-07-15' 
  AND CONVERT(datetime, CreateDate) < '2025-07-16'

-- 或者使用时分秒版本
WHERE CONVERT(datetime, CreateDate) >= '2025-07-15 00:00:00' 
  AND CONVERT(datetime, CreateDate) <= '2025-07-15 23:59:59'

-- 查询某月整月数据
WHERE CONVERT(datetime, CreateDate) >= '2025-07-01' 
  AND CONVERT(datetime, CreateDate) < '2025-08-01'
```

3. **常见错误场景：**
   - 用户说"查询7月15日的数据"，如果只用`<= '2025-07-15'`会漏掉当天下午和晚上的数据
   - 用户说"7月的数据"，结束条件应该是`< '2025-08-01'`而不是`<= '2025-07-31'`

4. **各日期字段适用：**
   - CreateDate, CloseDate, DeliveryDate, ConsumeDate 等所有日期字段都可能有此问题
   - 每次涉及日期范围查询时都要检查结束日期的处理方式

**供应商责任过滤逻辑（何时添加 Responsibility LIKE '%Supplier%' 条件）：**
为确保供应商绩效相关指标只统计供应商责任范围，需根据查询意图智能决定是否在WHERE中添加 `AND Responsibility LIKE '%Supplier%'`：
1. 以下场景应自动加入过滤条件（且排除 Pending/NULL/''）：
    - 供应商停线时间 / 实际停线时间（FixTime 汇总）
    - 供应商绩效/表现/打分/评分/改善建议/责任归属分析
    - 供应商 NCM 影响、供应商导致的 NCM 次数、供应商导致的问题数量
    - 查询描述中同时出现供应商标识（供应商代码或名称）与 FixTime / 停线 / 绩效 / 打分 / 责任 / 表现 等关键词
    示例："9400054 7月份停线时间"、"供应商9305665责任表现"、"FY25供应商打分"。
    SQL示例：
    ```sql
    SELECT SUM(CONVERT(decimal(7,2), FixTime)) AS 实际停线时间小时
    FROM ModelCapaSsme
    WHERE Supplier = '9400054'
       AND Responsibility LIKE '%Supplier%'
       AND CONVERT(datetime, CreateDate) >= '2025-07-01'
       AND CONVERT(datetime, CreateDate) < '2025-08-01';
    ```
2. 以下场景不加 Responsibility 过滤（保留全部责任方）：
    - 仅针对某个物料查询其 NCM 数量 / 物料出问题次数（需要统计该物料整体情况）
    - 需要展示/对比不同 Responsibility 的分布（例如按责任方分组或 Pending 列表）
    - 查询未体现供应商绩效语义，只是列出记录或做总体 NCM 统计。
    示例："物料1234567890的NCM次数"、"列出已关闭但责任未明确的NCM"。
    SQL示例：
    ```sql
    SELECT SUM(CONVERT(int, ActionCodeFactor)) AS NCM次数
    FROM ModelCapaSsme
    WHERE DefectivePartMaterialNumber = '1234567890'
       AND CONVERT(datetime, CreateDate) >= '2025-07-01'
       AND CONVERT(datetime, CreateDate) < '2025-08-01';
    ```
3. 责任未明确专门查询：应使用条件 `(Responsibility IS NULL OR Responsibility = '' OR Responsibility = 'Pending')`，同时结合 CloseDate 不为空场景。
4. 按责任方分布/对比：不强制加 `Responsibility LIKE '%Supplier%'`，而是 `GROUP BY Responsibility` 或在SELECT中按不同责任方分段聚合。
5. 如果用户语句含糊同时出现供应商与物料，可优先判断是否强调绩效/停线/打分；若无此类关键词则不加过滤。必要时可在结果说明中提示“已按供应商责任过滤”或“未过滤责任，包含所有责任方”。
